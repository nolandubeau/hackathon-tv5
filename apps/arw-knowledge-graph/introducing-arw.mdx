## The Agent-Web Interoperability Gap

AI agents are transforming how we interact with the web. ChatGPT's Atlas browser serves over 100 million weekly users. Perplexity's Comet powers millions of searches daily. Claude, Gemini, and autonomous agents help users research, shop, and complete tasks across thousands of websites.

**The reality:** AI agents already generate 40% of web traffic, with projections reaching 70% by 2027.

**The problem:** The web wasn't built for agents‚Äîit was built for human browsers.

Today, agents waste 85% of their bandwidth scraping HTML designed for visual rendering. They must crawl entire sites to discover capabilities. They can't complete transactions safely. And publishers have zero visibility into agent behavior or a standardized way to declare usage policies.

This isn't an adversarial problem‚Äîit's an infrastructure gap. Both agents and publishers would benefit from a standard layer for efficient, observable, and secure interaction.

**That's why we're building Agent-Ready Web (ARW).**

## What is ARW?

ARW is an open specification that provides the missing infrastructure layer for agent-web interoperability. It combines the discoverability of `llms.txt`, the access control of `robots.txt`, and the semantics of Schema.org into a unified, production-ready framework.

**The core innovation:** ARW separates *how content looks* (HTML for humans) from *how content is understood* (Markdown for agents), enabling both to coexist without compromise.

### Five Infrastructure Layers

**1. Efficient Content Layer**
- Machine views (`.llm.md` files) reduce tokens by 85% vs HTML
- 55KB HTML becomes 8KB Markdown‚Äîsame information, optimized format
- Semantic chunking enables precise citations and attribution

**2. Structured Discovery**
- Single manifest (`llms.txt`) lists all agent capabilities
- 10x faster than crawling entire sites
- `.well-known` JSON files for machine-optimized discovery
- Works with existing `sitemap.xml` and `robots.txt`

**3. Action Layer**
- OAuth-enforced transactions (technically protected, cannot be bypassed)
- Standard Schema.org semantics for declarative actions
- Safe agent operations: purchases, bookings, form submissions

**4. Observability Layer**
- `AI-*` headers track which agents visit and what they consume
- Foundation for agent analytics platforms
- Visibility into agent behavior and value

**5. Policy Declaration**
- Machine-readable usage terms (training vs inference)
- Advisory declarations (like `robots.txt`) with legal foundation
- Standardized attribution requirements and rate limits

## Where We Are: ARW v1.0 Release Candidate

After months of development, research, and refinement, **ARW v1.0 is now in Release Candidate status**. Here's what's ready today:

### Production-Ready Specification

The [ARW v1.0 specification](https://github.com/nolandubeau/agent-ready-web/blob/main/spec/ARW-v1.0.md) defines four conformance levels:

- **ARW-1 Discovery**: Basic discovery via `llms.txt` + machine views
- **ARW-2 Semantic**: Adds chunking, attribution, rate limits
- **ARW-3 Actions**: OAuth actions + Schema.org semantics
- **ARW-4 Protocol**: Protocol interoperability (MCP, ACP, A2A)

Each level builds progressively‚Äîimplement incrementally without breaking existing functionality.

### Production-Ready Tooling

**Rust CLI Tool** ([`packages/cli/`](https://github.com/nolandubeau/agent-ready-web/tree/main/packages/cli))
- 100% ARW-1 conformance, 95% ARW-2 conformance
- Commands: `init`, `generate`, `validate`, `serve`, `scan`, `watch`
- 100% JSON Schema conformance validation
- WASM compilation support for browser/Node.js
- 35+ tests, 45%+ coverage
- Ready for NPM distribution as `@agent-ready-web/cli`

**Validation Tools** ([`packages/validators/`](https://github.com/nolandubeau/agent-ready-web/tree/main/packages/validators))
- Python validator: `validate-arw.py` (jsonschema + pyyaml)
- Node.js validator: `validate-arw.mjs` (ajv + ajv-formats)
- CI/CD integration ready
- VS Code and JetBrains editor integration

**Visual Inspector** ([`apps/arw-inspector/`](https://github.com/nolandubeau/agent-ready-web/tree/main/apps/arw-inspector))
- Browser-based inspection tool
- Side-by-side machine view comparison
- Chunk validation and identification
- URL input for live website testing

**Chrome DevTools Extension** ([`apps/arw-devtools-extension/`](https://github.com/nolandubeau/agent-ready-web/tree/main/apps/arw-devtools-extension))
- Native DevTools ARW inspection panel
- Real-time validation
- Works on localhost (no CORS issues)

## The Economic Argument

ARW isn't optional infrastructure‚Äîit's economically essential.

Recent NBER research on the ["Coasean Singularity"](https://arw.dev/blog/coasean-singularity-arw-infrastructure) shows that AI agents reduce transaction costs toward near-zero, fundamentally reshaping markets. ARW enables this transformation:

- **Search costs**: ‚Üì 90% via structured discovery
- **Communication costs**: ‚Üì 85% via machine views
- **Contracting costs**: ‚Üì 80% via declarative actions
- **Verification costs**: ‚Üì 90% via observability headers

When transaction costs approach zero, agent-mediated commerce becomes viable at scale. ARW is the infrastructure that makes this possible.

## Two Enforcement Models

ARW recognizes that different use cases require different approaches:

**Technical Enforcement (OAuth Actions)**
- Cannot be bypassed by agents
- Required for transactions: purchases, bookings, data mutations
- Standard OAuth 2.0 flows with user consent
- Examples: `add_to_cart`, `checkout`, `submit_support_ticket`

**Advisory Declarations (Content Policies)**
- Like `robots.txt`‚Äîadvisory but with legal foundation
- Provide observability (track compliance via headers)
- Enable analytics (measure agent behavior)
- Create platform leverage (basis for AI company commitments)
- Examples: training permissions, attribution requirements, rate limits

This dual approach recognizes reality: transactional operations can be technically protected, while content access benefits from transparency and standardization.

**2. Publishers Need Infrastructure**
